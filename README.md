# Forge (working title)

Structured strength programming + training-load-adaptive nutrition. See `../prd.html`, `../engineering-backlog.html`, and `../Wireframes/` for product context.

## Structure

- `Sources/ForgeCore` ‚Äî pure Swift business logic (Load Score, TDEE, macro/calorie engine). No UIKit/SwiftUI dependency, builds and tests with just the Swift toolchain ‚Äî no Xcode required.
- `Tests/ForgeCoreTests` ‚Äî Swift Testing (`import Testing`) coverage for the engine above.
- `App/` ‚Äî the SwiftUI iOS app. Requires Xcode to build.
- `project.yml` ‚Äî [XcodeGen](https://github.com/yonaskolb/XcodeGen) spec. The `.xcodeproj` is generated, not committed ‚Äî regenerate after adding files.
- `FoodProxy/` ‚Äî a small Node/Express server that proxies FatSecret API calls (see its README for why: FatSecret requires a fixed, allowlisted IP and server-held credentials, which a mobile app can't provide directly). Deployed separately (Render), not part of the Xcode project.

## Getting started

Xcode is installed on this machine. XcodeGen isn't available via Homebrew here (Homebrew itself isn't installed), so it was built from source once ‚Äî if you need to regenerate the project after adding files and don't have `xcodegen` on PATH, either `brew install xcodegen` or build it yourself:

```
git clone --depth 1 https://github.com/yonaskolb/XcodeGen.git /tmp/XcodeGen
cd /tmp/XcodeGen && swift build -c release --product xcodegen
```

Then from this folder: `xcodegen generate`, open `Forge.xcodeproj`, select a simulator, and run.

To run just the engine tests: `swift test`.

**How logging reminders, Apple Health sync, and data export actually work** (asked, answered here rather than as a code change):
- **Logging reminders** (`ReminderManager`, You tab toggle): local notifications only, no server/push infra. Turning it on requests notification permission once; `AppStore.refreshReminders()` (called on toggle-enable and every app backgrounding) schedules a 7pm "log your workout" and 8pm "log your food" local notification *only* for whichever of those two you haven't done yet that day, and cancels each one the moment you actually do it (`toggleSet`/`logFood` call `ReminderManager.shared.cancelWorkoutReminder()`/`cancelMealReminder()` directly). Nothing fires if you've already logged both.
- **Apple Health sync** (`HealthKitManager`, You tab toggle): read-only HealthKit authorization for steps and sleep. Enabling it requests the OS permission dialog once, then calls `AppStore.syncHealthKit()`, which populates `stepsToday`/`lastNightSleepHours` ‚Äî these feed FRG-305's sleep-based Load Score dampening (poor sleep pulls back an already-elevated day, never adds calories) and are shown as a small "steps today / sleep last night" readout right under the toggle. It's a fresh query each time (on toggle-enable, and again on every YouView visit if already on), not a live subscription ‚Äî closing and reopening the app is what refreshes it, not a background push.
- **Export data (CSV)** (`CSVExporter`, You tab): builds CSV files in memory and hands them to the system share sheet (`ShareLink`) ‚Äî AirDrop, Files, Mail, whatever the user picks. Bodyweight history and per-session workout volume export in full; nutrition/exercise-level detail is limited to *today* specifically, since the in-memory store doesn't hold multi-day food/exercise detail beyond what's already loaded from CloudKit ‚Äî historical CloudKit records exist, but this exporter doesn't currently page back through them.
